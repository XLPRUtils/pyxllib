# pyxllib 文件处理模块设计精华总结

> 本文档总结了 `pyxllib.file.specialist.filelib` 和 `dirlib` 中值得保留和借鉴的优秀设计模式。在迁移到标准 `pathlib` 时，这些设计思想可以作为扩展工具库或最佳实践的参考。

## 1. 核心设计哲学

*   **智能与自动化 (Smart & Auto)**: 尽量减少用户的参数输入，自动推断编码 (`charset_normalizer`)、文件格式 (`.json`, `.pkl`) 和操作意图。
*   **安全优先 (Safety First)**: 所有的写操作 (Write/Copy/Move) 都通过 `if_exists` 机制保护，支持自动备份 (`backup`)，避免意外覆盖。
*   **批量处理 (Batch Processing)**: `Dir` 类采用 "Select-Process" 模式，先通过复杂的过滤器选中一批文件，再统一执行操作。

## 2. `filelib.py` 亮点功能

### 2.1 智能 I/O (`File.read`, `File.write`)
封装了底层的 `open` 和序列化库，根据文件后缀自动选择策略。

*   **自动编码识别**: 读取文本时若未指定编码，使用 `charset_normalizer` 自动检测。
*   **格式感知**:
    *   `.json`: 自动 `json.load/dump`，且 `write` 默认 `ensure_ascii=False` (对中文友好)。
    *   `.pkl`: 自动 `pickle` 处理。
    *   `.yaml`: 自动 `yaml` 处理。
*   **代码参考**: `read_auto` 和 `write_auto` 方法。

### 2.2 安全写操作 (`exist_preprcs`)
在执行写操作前，统一检查目标文件状态。
*   **机制**: `if_exists` 参数支持：
    *   `'error'`: 报错。
    *   `'skip'`: 跳过。
    *   `'replace'`: 删除旧文件。
    *   `'backup'`: **自动将旧文件重命名**（添加时间戳后缀），这是非常实用的工程特性。

### 2.3 增强的 Path 对象 (`XlPath`)
继承自 `pathlib.Path`，但补充了实用方法：
*   **`safe_init`**: 初始化失败不报错而是返回 `None`，适合处理脏数据路径。
*   **`size`**: 统一了文件大小和**目录递归大小**的获取接口。
*   **`res = p.sub_rel_paths`**: 快速获取目录下所有文件的相对路径集合（用于目录对比）。
*   **`split_to_dir` / `merge_from_files`**: 针对大文件的拆分与合并工具，适合处理日志或大数据集。

### 2.4 实用工具
*   **`GetEtag`**: 计算文件的 ETag（基于七牛算法），比 MD5 更适合云存储场景的大文件校验。
*   **`cache_file` 装饰器**: 极其好用的函数结果缓存工具。如果文件存在则直接读取，不存在则执行函数并保存文件。支持过期时间 (`cache_time`)。
*   **`StreamJsonlWriter`**: 流式写入 JSONL，带有缓存 buffer，适合处理超大规模数据写入。

## 3. `dirlib.py` 亮点功能

### 3.1 强大的文件筛选 (`filesmatch`, `Dir.select`)
比标准 `glob` 更强大的筛选器。
*   **多模式支持**: `patter` 可以是列表 `['*.jpg', '*.png']`，一次性匹配多种后缀。
*   **正则支持**: 支持传入 regex 对象进行文件名匹配。
*   **属性过滤**: 在 glob 的同时，支持按 `min_size`, `min_mtime` (时间范围) 进行过滤。
*   **排除机制**: `Dir.exclude` 支持反向筛选。

### 3.2 目录重组工具
针对深度学习数据集清洗非常实用的功能：
*   **`flatten_directory`**: 拍平目录，将所有子目录文件移到根目录（处理嵌套过深的脏数据）。
*   **`nest_directory`**: 目录分片，将大量文件按数量或 stem 分组存入子目录（避免单文件夹文件数过多）。
*   **`refine_files_suffix`**: 基于文件头 (`filetype`) 修正错误的文件后缀（如将 `.jpg` 实际上是 png 的文件改名）。

### 3.3 批量重命名与去重
*   **`BatchFileRenamer`**: 处理文件名冲突的逻辑（自动添加 `(1)`, `(2)` 后缀）。
*   **`check_repeat_files`**: 基于 ETag 查找目录下的重复文件，并支持保留一份、删除其他的操作。

## 4. 迁移建议

在移除 `pyxllib` 依赖时，建议：

1.  **保留 `safe_io` 逻辑**: 封装一个 `write_json(path, data, if_exists='backup')` 的独立函数，保留备份机制。
2.  **保留 `cache_file`**: 这是一个独立的、高价值的通用工具，建议移至项目的 `utils` 模块。
3.  **替换 `Dir`**: 使用 `pathlib.Path.rglob` 结合列表推导式替代 `Dir.select`。
    *   *旧*: `Dir('imgs').select('*.jpg', min_size=1024)`
    *   *新*: `[p for p in Path('imgs').rglob('*.jpg') if p.stat().st_size > 1024]`
4.  **替换 `XlPath`**: 直接使用 `pathlib.Path`，对于 `read_text(encoding='auto')` 的需求，建议显式使用 `charset_normalizer` 或统一项目编码为 UTF-8。
